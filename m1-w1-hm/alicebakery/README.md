2Q. Make a list of scenarios where you feel Spring Framework can be very useful over NodeJS. 
Ans:: (NO Idea about NodeJs)
1. Large Enterprise Applications - Spring Framework is widely used for large-scale enterprise applications because of its strong architecture and stability.
2. Complex Business Logic - Spring handles complex business logic and layered architecture very effectively.
3. Security Features - Spring Security provides powerful authentication and authorization features for secure applications.
4. Transaction Management - Spring provides excellent database transaction management, especially for banking and financial systems.
5. Microservices and Integration - Spring works very well with microservices, distributed systems, and enterprise integrations.
6. Strong Type Safety - Since Spring uses Java, it provides compile-time checking and better type safety compared to JavaScript-based NodeJS

3Q. Make a list of scenarios where you feel Spring Boot can be very useful over Spring Framework.
Ans::
1.Embedded Tomcat Configuration - Spring Boot provides an embedded Tomcat server, so we do not need to configure and deploy the application manually on an external server.
2.Auto-Configuration - Spring Boot automatically configures dependencies and project setup based on the dependencies added in pom.xml or build.gradle.
3. Easy Bean Management - Spring Boot simplifies bean creation, initialization, and dependency injection using annotations and automatic component scanning.
4. Faster Development - Spring Boot reduces boilerplate code and configuration, helping developers build applications quickly.
5. Production-Ready Features - Spring Boot provides built-in features like health checks, metrics, and monitoring through Spring Boot Actuator.
